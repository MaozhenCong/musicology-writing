# Journal and Thesis Adaptation

Use this reference when a manuscript must fit a specific journal, OJS portal, email submission, doctoral requirement, or thesis chapter requirement.

## Target Requirement Audit

Create a target audit before formatting:

```markdown

```

Only use official pages, author guidelines, OJS pages, journal PDFs, editor emails, or supervisor/doctoral-office documents as requirement evidence.

## Journal-Fit Adaptation

Adapt the same research area differently depending on target:

- Music education journal: foreground pedagogy, curriculum, teacher training, didactics, learning context, institutional practice.
- Musicology journal: foreground reception, historiography, repertoire, cultural mediation, source interpretation, musicological contribution.
- Intercultural/cross-cultural journal: foreground mediation, translation, institutional circulation, transnational reception, conceptual framing.
- Doctoral thesis chapter: foreground sustained literature review, method transparency, relation to the dissertation argument, and supervisor/degree rules.

Do not force a journal to accept a frame it does not publish. If the best article for the evidence does not fit the target, say so.

For detailed track rules, use `references/dissertation-chapter-workflow.md` for thesis chapters and `references/journal-article-workflow.md` for journal articles.

## Dissertation Format Adaptation

For thesis chapters:

1. Confirm the required citation style and output format.
2. Use DOCX as the default final deliverable when the user wants an editable dissertation chapter.
3. If Chicago Notes-Bibliography is requested for a dissertation, apply it consistently across the whole dissertation: place one note callout at the citation position and place multiple sources in that single note with semicolons.
4. Cite verified printed pages from the original source when available; use `file p.` only when printed pagination cannot be confirmed or does not exist after inspection.
5. Use rendered pages or Word inspection for layout QA, but do not deliver a final PDF unless requested.
6. Preserve breadth appropriate to dissertation work while keeping every claim traceable to evidence.
7. Treat margin, heading, footnote-font, page-numbering, front-matter, and institutional metadata rules as external thesis requirements; verify them from supervisor or university documents before final formatting.

## Fast Doctoral Publication Mode

When the user needs a publication for doctoral requirements:

1. Favor journals that are officially acceptable, open now, topically compatible, and procedurally clear.
2. Keep the topic narrow enough to finish and defend quickly.
3. Avoid new interviews or ethics-dependent data unless already approved.
4. Save proof of official list status, ISSN, issue/year, deadline, fees, and editor correspondence.
5. Treat final publication as the safe requirement unless the doctoral office confirms acceptance or online-first is enough.

## Format and Submission Package

Depending on the target, prepare:

- anonymous manuscript;
- authored manuscript or title page;
- abstract and keywords;
- cover letter or submission email;
- OJS copy/paste metadata;
- author biography;
- funding, conflict, data, ethics, AI-use, and contribution statements;
- similarity report if required;
- response-to-reviewers file for revisions;
- final local submission record.

Do not invent production metadata such as DOI, received date, accepted date, volume, issue, or copyright line. Those belong to the publisher unless already assigned.

## Post-Submission State Changes

After submission:

1. Mark the lane as submitted or under review only when there is evidence: OJS acknowledgement, editor email, or sent-mail record.
2. Record exact IDs, dates, file names, email message IDs if available, and dashboard URLs.
3. Freeze the submitted files.
4. Do not replace the manuscript unless the journal asks.
5. If revision arrives, copy the submitted version into a new revision workspace and create a response table before editing.

## Response to Reviewers

For each comment:

- quote or summarize the reviewer comment accurately;
- classify as required, optional, misunderstanding, or out of scope;
- decide action: revised, clarified, added evidence, declined with reason;
- point to exact manuscript location;
- keep tone professional and concise.

Basic response pattern:

```text
We thank the reviewer for this helpful comment. We have revised the manuscript to clarify [issue]. The revised text now [specific change], and the change appears in [section/page].
```

If disagreeing, acknowledge the issue, explain the evidence boundary, and offer a smaller clarification.
