# Journal Article Workflow

Use this reference for musicology/music education journal articles, fast doctoral-publication articles, OJS submissions, email submissions, revisions, and under-review monitoring.

## Track Rule

Use the journal track when the output is a publishable article, target-journal package, response to reviewers, cover letter, OJS metadata, anonymous manuscript, or submission-state update. Do not import dissertation-chapter habits such as broad survey coverage, long footnote clusters, or thesis-scale framing unless the target journal welcomes them.

## Lane Isolation

Keep each journal lane separate:

- one target journal per status record;
- one source matrix and evidence log per article;
- one submitted-file archive per submission;
- one vocabulary, thesis, and corpus per article to reduce similarity risk.

Do not mix active journals, submitted versions, or editor correspondence across lanes unless the user asks for comparison.

Preferred lane layout:

```text
active_journals/
└── journal_slug/
    ├── status.md
    ├── source_matrix.tsv
    ├── evidence_log.tsv
    ├── submission_record.md
    ├── draft/
    ├── submitted/
    └── revision/
```

Follow existing project names when present, but preserve the separation between draft, submitted, and revision files.

## Target Audit

Before drafting or finalizing, verify official requirements from journal pages, OJS pages, author guidelines, saved PDFs, or editor email:

- scope and article type;
- language;
- word/page length;
- citation style;
- anonymity and metadata;
- file format;
- declarations and AI-use policy;
- fees, deadlines, and issue timing;
- submission route.

If a requirement is not visible, mark it unverified instead of guessing.

## Fast Doctoral Publication Mode

When publication is needed for doctoral progress:

1. Prefer a narrow topic that can be fully evidenced quickly.
2. Prioritize official eligibility, fit, procedural clarity, and current calls.
3. Avoid new ethics-dependent data unless approvals already exist.
4. Save evidence of official list status, deadline, fee policy, and submission route.
5. Treat acceptance or online-first as sufficient only if the doctoral rule confirms it.

## Citation and Footnote Handling

For journal articles, keep one source per footnote by default during drafting and audit because it improves traceability. Merge multiple sources into one note only when:

- the journal style prefers combined notes;
- the user asks for merged notes;
- the final citation audit confirms the merged note remains clear.

Still cite exact pages for paginated sources. If a target journal uses author-date style, convert only after the evidence audit so page locators are not lost.

## Similarity-Risk Control

Before submission:

- compare title, abstract, headings, and key phrases against the user's dissertation and other submitted articles;
- change corpus, framing, and central vocabulary for adjacent article lanes;
- avoid reusing the same introduction architecture across journals;
- do not copy attractive paragraphs from a dissertation chapter without rewriting and re-evidencing them;
- make overlap transparent if the journal requires disclosure.

## Final Package

Prepare only what the target requires:

- anonymous manuscript;
- authored manuscript or title page;
- abstract and keywords;
- cover letter or submission email;
- OJS metadata;
- declarations;
- author bio;
- response-to-reviewers file if revising;
- local submission record.

Do not invent DOI, volume, issue, received date, accepted date, or production metadata.

## Submission and Monitoring

After submission:

1. Mark the lane as submitted or under review only with evidence: acknowledgement email, sent email, OJS dashboard, or editor reply.
2. Record submission ID, date, file names, message ID if available, dashboard URL, and current status.
3. Freeze the submitted version.
4. Do not replace files unless the journal requests it.
5. If reviews arrive, create a response table before editing.
